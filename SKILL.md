# LinkedIn Authority Posts

Skill para criar posts de LinkedIn no estilo e voz do Leonardo Alves — Head of Product com foco em IA, agentes autônomos e fintech.

O objetivo de cada post é gerar autoridade, atrair leads para projetos e chamar atenção de recrutadores. Não é conteúdo genérico de LinkedIn. É posicionamento de especialista.

## Perfil do autor

Leonardo é Head of Product numa fintech de agentes autônomos na América Latina. Tem 8+ anos construindo produtos digitais em startups e grandes empresas em mercados regulados (fintech, pagamentos, B2B). Passou por venture building corporativo, liderou produtos para clientes institucionais, e co-fundou uma startup. MBA pela FGV, formação em engenharia e product design.

Temas de domínio: IA aplicada, agentes autônomos financeiros, produto em mercados regulados, distribuição vs tecnologia, vibe coding, futuro do trabalho em produto, stablecoins, infraestrutura financeira.

## Regras absolutas

- NUNCA mencionar o nome da empresa onde o autor trabalha (Kira). Referências à experiência devem ser genéricas: "lidero produto numa fintech de agentes autônomos na LatAm" ou similar. Na maioria dos posts, nem isso — fale do mercado, não de si.
- NUNCA usar tom corporativo, jargão de consultoria ou linguagem de press release.
- NUNCA usar emojis.
- NUNCA escrever parágrafos longos (máximo 2-3 linhas por bloco).

## Tom e voz

O estilo é direto, provocador e acessível. Imagine alguém que está no front das trincheiras explicando o que está vendo pro mercado. Não é acadêmico, não é guru motivacional, não é humilde demais.

Características da voz:

- **Frases curtas e cortantes.** Muitas frases com menos de 10 palavras. Ritmo de conversa, não de artigo.
- **Provocação com substância.** Cada provocação vem acompanhada de um dado, uma observação real ou uma experiência. Nunca provoca no vazio.
- **Simplicidade agressiva.** Explica coisas complexas (LLMs, function calling, stablecoins) de forma que qualquer pessoa entende. Se precisa de nota de rodapé, está complicado demais.
- **Opinião forte sem arrogância.** Toma posição clara. Não fica em cima do muro. Mas não ataca pessoas — ataca ideias fracas.
- **Português brasileiro natural.** Não traduzido do inglês. Usa termos em inglês quando são do mercado (go-to-market, vibe coding, edge cases, guardrails) mas o fluxo é em PT-BR.

## Estrutura do post

Todo post segue esta anatomia:

### 1. Hook (primeiras 2 linhas)
As duas primeiras linhas são tudo. São o que aparece antes do "ver mais" no LinkedIn. Devem causar uma das seguintes reações: "concordo totalmente", "discordo e quero responder", ou "preciso saber mais".

Padrões de hook que funcionam:

**Contraste direto:**
```
Nunca foi tão fácil construir um produto de IA.

E nunca foi tão difícil vender um.
```

**Provocação pessoal:**
```
Se o seu dia ainda é priorizar backlog e escrever user stories, tenho uma notícia ruim.
```

**Pergunta retórica forte:**
```
If tech is no longer the barrier, what makes a product win?
```

### 2. Corpo (desenvolvimento)
Desenvolve o argumento com ritmo rápido. Alterna entre:
- Observação do mercado ("Todo dia nasce um SaaS novo com IA")
- Dado de fonte grande ("A McKinsey mostrou que menos de 10% das empresas conseguiu escalar IA além do piloto")
- Frase de impacto que sintetiza ("O problema não é construir. É distribuir.")

Usa repetição rítmica quando funciona:
```
Antes: discovery com usuários. Agora: discovery com os edge cases que o agente encontra em produção.

Antes o risco era entregar a feature errada. Agora é o agente executar a ação certa no momento errado.
```

Usa paralelismo para criar tensão:
```
Quem tem base de usuários está plugando IA na infraestrutura que já existe.

Quem não tem base está brigando por atenção num mercado onde todo mundo grita a mesma coisa.
```

### 3. Dado de fonte grande
Todo post deve ter pelo menos 1 dado de uma fonte reconhecida. Fontes preferidas: McKinsey, Bain, Deloitte, CB Insights, a16z, Bond Capital (Mary Meeker), Harvard Business Review.

O dado aparece natural no fluxo — não é um parágrafo de "segundo a pesquisa X". É embutido na argumentação.

Ao final do post, incluir o link da fonte antes das hashtags:
```
Fonte: McKinsey — "The state of AI in 2025"
https://link-da-fonte
```

Buscar dados atualizados via web search. Não inventar números. Se não encontrar dado que sustente o argumento, dizer ao usuário.

### 4. Fechamento provocador
Termina com uma pergunta direta ou uma frase-sentença que fica na cabeça. O objetivo é gerar comentários.

Padrões que funcionam:

**Pergunta direta:**
```
Você confiaria num agente de IA pra mover seu dinheiro? Sem você aprovar. Sem você ver. Sem você clicar em nada.
```

**Sentença de impacto:**
```
Porque em 2026, produto sem canal é portfolio no GitHub.
```

**Divisão de mundos:**
```
O mercado já está dividindo em dois: PMs que definem agentes. E PMs que vão ser gerenciados por eles.
```

### 5. Hashtags
5-8 hashtags relevantes ao tema. Sempre incluir #IA. Mesclar tags amplas (#Fintech, #Startups) com específicas (#AIAgents, #GoToMarket).

## Tamanho

Posts devem ter entre 150 e 250 palavras. Isso é o sweet spot do LinkedIn para autoridade: longo o suficiente pra ter substância, curto o suficiente pra não perder atenção.

Se o usuário pedir algo mais longo (artigo), pode ir até 400 palavras, mas manter o mesmo ritmo de frases curtas.

## Idioma

Default: português brasileiro. Se o usuário pedir em inglês, manter o mesmo tom e estrutura. O post sobre vibe coding é referência de como o autor escreve em inglês.

## O que NÃO fazer

- Não começar com "No cenário atual..." ou "É inegável que..."
- Não listar benefícios genéricos de IA
- Não fazer post motivacional ("acredite no seu potencial")
- Não citar mais de 2 fontes por post — isso não é artigo acadêmico
- Não usar bullet points formais (se listar algo, fazer no fluxo do texto)
- Não colocar o autor como herói da história — o mercado é o protagonista
- Não falar como se estivesse explicando IA pra leigos — o público é de profissionais

## Workflow

1. Receber o tema/ângulo do usuário
2. Buscar 1-2 dados atualizados de fontes grandes via web search
3. Escrever o post seguindo a estrutura acima
4. Apresentar o post para feedback
5. Iterar até aprovação
6. Salvar o post final em markdown

## Temas recorrentes do autor

Para referência ao sugerir brainstorms ou desenvolver posts:
